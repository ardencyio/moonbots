# Fix 001: Open Gate Strategy Backtest Integration

## Issue Summary

Backtest runs and generates trades, but TP calculation is incorrect because `position_direction` was reportedly not set before calling `calculate_take_profit()`. After review, the code already sets `position_direction` before the call — but two latent bugs remain that can cause desync and incorrect TP on subsequent trades.

---

## Root Causes

### Bug 1: `position_direction` not reset on close

`_close_position()` clears `position_open` but **not** `position_direction`:

```python
# Current (open_gate.py:327-332)
self.position_open = False
# position_direction is NOT cleared — stale value persists
```

This leaves a stale direction that can corrupt TP calculation on the next trade.

### Bug 2: `position_open` desyncs from backtesting.py's `self.position`

In `backtesting_runner.py:54-58`, the runner checks `if self.position:` then calls `strategy.check_exit()`. But `check_exit()` relies on `self.position_open` — a separate flag that can diverge from backtesting.py's internal position state. When they diverge, exit checks get skipped.

---

## Fixes

### Fix 1 — `open_gate.py` line ~329

Add `position_direction = None` reset in `_close_position()`:

```python
# Reset position
self.position_open = False
self.position_direction = None  # ← ADD THIS
self.entry_price = None
self.stop_loss = None
self.take_profit = None
```

### Fix 2 — `backtesting_runner.py` lines ~54-58

Sync `position_open` flag when a backtesting.py position exists:

```python
if self.position:
    self.strategy.position_open = True  # sync flag
    exit_signal = self.strategy.check_exit(current_row)
    if exit_signal:
        self.position.close()
        return
```

---

## Verification

Run a backtest and verify:
1. Trades have correct long/short TP levels (long TP > entry, short TP < entry)
2. No skipped exits (all trades close with a reason)
3. `position_direction` is `None` between trades
