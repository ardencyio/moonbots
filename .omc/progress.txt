  the richer regime_infos loaded from the pickle (Phase-3 fields beyond the
  four _label_regimes writes). Recomputing the lexsort directly in load()
  keeps regime_infos intact.

  training guarantees; the test helper asserts `engine.model is not None`
  before touching .means_ to satisfy the narrowing.

### Learnings / Gotchas:
- Use `uv run python -m pytest` (not `uv run pytest`) so the root conftest runs and
  `shared.*` imports resolve.
- hmmlearn exposes only `predict_proba` (smoothed) publicly; to verify filtered
  (forward-only) output, use a hand-rolled forward recurrence with
  `model._compute_log_likelihood` + `scipy.special.logsumexp`.
- 4-regime vol-rank bucketing: positions [0.0, 1/3, 2/3, 1.0] with thresholds
  <=0.33 / <=0.66 map to low/mid/high — regime at index 1 (vol_fraction=1/3) is
  strictly > 0.33 so it falls into mid; regime at index 2 (vol_fraction=2/3 ≈ 0.6667)
  is strictly > 0.66 so it falls into high. Test expectations must follow this
  partition rather than uniform quartiles.
- scipy.linalg.LinAlgError vs np.linalg.LinAlgError are distinct exception classes;
  mahalanobis solve must catch both plus ValueError for singular covariance matrices.
